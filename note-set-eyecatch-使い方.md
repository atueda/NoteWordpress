# note-set-eyecatch.js 使い方ガイド

WordPressのアイキャッチ画像を、Noteの記事に自動でアップロードするスクリプトです。

---

## このスクリプトでできること

WordPress記事に設定されたアイキャッチ画像を、自動的にNoteの対応する記事にアップロードします。

- Note一覧画面で「画像なし」の記事だけを対象にします（すでに画像がある記事はスキップされます）
- 途中で止めても、次回は続きから再開できます

---

## 前提条件（実行前に必ず確認）

### Mac専用です

このスクリプトは **Macでのみ動作します**。Windowsでは動作しません。

### 実行前に必要な作業

このスクリプトはプロジェクト全体の **STEP 7（最後のステップ）** です。  
以下がすべて完了していることを確認してください。

| 完了させるもの | 確認方法 |
|---|---|
| `node wp-to-note.js` を実行済み | `note-export/_index.txt` ファイルが存在する |
| `node note-auto-post.js` を実行済み | Noteに記事が下書き投稿されている |
| `node note-publish-one.js` を実行済み | Noteの記事が公開されている |
| Noteにブラウザでログイン済み | `note-browser-profile/` フォルダが存在する |

### Note一覧でアイキャッチ画像がないことを確認

Noteの記事一覧ページ（https://note.com/notes）を開いて、アイキャッチ画像がまだ設定されていない記事があることを確認してください。

---

## 実行手順

### 手順1：ターミナルを開く

Finderで「アプリケーション」→「ユーティリティ」→「ターミナル」をダブルクリックして開きます。

または、Spotlight検索（`Cmd + スペース`）で「ターミナル」と入力して開きます。

### 手順2：プロジェクトフォルダに移動する

ターミナルに以下をそのままコピー＆ペーストして、Enterキーを押します。

```
cd ~/NoteWordpress
```

### 手順3：スクリプトを実行する

以下をコピー＆ペーストして、Enterキーを押します。

```
node note-set-eyecatch.js
```

---

## 実行中の動作

スクリプトを起動すると、以下のことが自動で行われます。

1. **ChromiumブラウザがNote一覧（https://note.com/notes）を開きます**
2. **各記事について順番に処理します：**
   - WordPress REST APIから記事のタイトルとアイキャッチ画像を取得
   - Note一覧でその記事を探し、アイキャッチ画像の有無を確認
   - 画像がすでにある場合 → **スキップ**
   - 画像がない場合 → **エディタを開いて画像をアップロードし、更新（公開）する**
3. **すべての記事を処理したら終了します**

ブラウザウィンドウが自動で開いて操作されますが、処理中は触らずにそのままにしておいてください。

---

## 進捗の確認と再開

処理の進捗は `eyecatch-progress.csv` というファイルに自動保存されます。

- 途中でスクリプトを止めた（`Ctrl + C`）場合も、次回実行時は **未処理の記事から続きを再開** します
- 「done（完了）」と記録された記事は二重処理されません

---

## テスト実行（1件だけ試したいとき）

いきなり全件処理するのが不安な場合は、スクリプトを開いて `maxCount` の設定を変更してください。

`note-set-eyecatch.js` をテキストエディタで開き、以下の部分を探します。

```javascript
  maxCount: Infinity,
```

これを以下に変更して保存します。

```javascript
  maxCount: 1,
```

これで1件だけ処理されます。確認できたら `Infinity` に戻してから全件実行してください。

---

## よくあるエラーと対処法

### 「ブラウザが起動しない」「セッションエラー」が出る

前回の実行が正常に終了しなかった可能性があります。  
以下のコマンドをターミナルで実行して、残っているプロセスを終了してから再実行してください。

```
pkill -f "note-browser-profile"
```

その後、もう一度 `node note-set-eyecatch.js` を実行してください。

### 「_index.txtが見つかりません」というエラーが出る

`note-export/_index.txt` が存在しない場合に起きます。  
先に `node wp-to-note.js` を実行して、ファイルを生成してください。

### Noteにログインが必要と表示される

ブラウザが自動で開いたら、手動でNoteにログインしてください。  
ログイン後、スクリプトは自動で処理を再開します。

## 設定の変更（上級者向け）

`note-set-eyecatch.js` の冒頭にある `config` の部分で動作を変更できます。

```javascript
const config = {
  publishMode: 'publish',  // 'draft' にすると下書き保存のみ（公開しない）
  maxCount: Infinity,      // 数字にすると処理件数を制限できる（例: 5 なら5件まで）
};
```

| 設定項目 | 説明 |
|---|---|
| `publishMode: 'publish'` | 画像設定後、そのまま更新（公開）する（デフォルト） |
| `publishMode: 'draft'` | 画像を設定して下書き保存するだけ（公開しない） |
| `maxCount: Infinity` | すべての記事を処理する（デフォルト） |
| `maxCount: 1` | 1件だけ処理して止まる（テスト用） |
